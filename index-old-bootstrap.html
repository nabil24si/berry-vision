System Architecture & Flowchart

This system classifies berry images using Convolutional Neural Network (CNN) technology. Below is the comprehensive workflow from data collection to deployment.

1

Dataset Engineering

Collection of berry images, data understanding, preprocessing (resizing to uniform format for model compatibility)

  • Gather images of Strawberry, Blueberry, and Blackberry
  • Perform exploratory data analysis (EDA)
  • Standardize image dimensions to 224x224 pixels
  • Data augmentation and normalization
2

Deep Learning Model

Building basic CNN architecture with convolutional and pooling layers

  • Design CNN model without transfer learning
  • Layers: Conv → ReLU → MaxPool → Flatten → Dense
  • Optimizer: Adam | Loss: Categorical Cross-Entropy
  • Training on GPU with batch processing
3

Backend System

Image processing logic, MVC architecture, and API communication with CNN model

  • Python Flask/FastAPI server
  • RESTful API endpoints for predictions
  • Model serving and inference
  • Request validation and error handling
4

Frontend System

Interactive dashboard interface for users to upload images and receive predictions

  • Responsive HTML5/Bootstrap interface
  • Real-time image preview before analysis
  • Interactive charts and statistics
  • Prediction result visualization
5

Deployment

System implementation on local server or cloud platform for user accessibility

  • Deploy on local server or cloud (AWS, GCP, Heroku)
  • Environment configuration and setup
  • API endpoint exposure and documentation
  • Monitoring and logging system
